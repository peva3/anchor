# 03-pr-standards.md

## PR Size Limits
- All changes: 800 lines max
- Complex logic: 500 lines max
- Single feature per PR — never mix feature + refactor

## PR Template
- Use the standard PR template with HUMAN/AGENT sections
- Include: What Changed, Why, How, Testing, Breaking Changes
- NEVER cite line numbers, list files, or include code snippets in description

## First-Time Contributions
- First PR should be a small bug fix (under 200 lines)
- Builds trust and demonstrates understanding of conventions
